# Anderson Espindola

Senior Software Engineer focused on Functional Programming, backend systems, and distributed architectures.

I build production software with JavaScript and TypeScript, and I am currently investing heavily in Functional Programming as my main engineering direction, especially through Elixir, Scala, and Haskell.

My goal is simple: design software with better models, fewer incidental side effects, stronger composition, and clearer reasoning.

## Current Focus

- Functional Programming as a default way of thinking
- Elixir for concurrent, fault-tolerant, message-oriented systems
- Scala for typed functional design on the JVM
- Haskell for precision, algebraic modeling, and deeper reasoning
- JavaScript and TypeScript written with a functional style in real-world production systems

## How I Like to Build Software

- Pure functions where possible, side effects isolated where necessary
- Immutability, composition, and explicit data flow
- Domain modeling with clear boundaries and meaningful abstractions
- Declarative transformations instead of imperative orchestration
- Correctness, testability, and maintainability before accidental complexity
- Performance and scalability treated as design concerns, not afterthoughts

## Functional Programming Interests

- Pattern matching and exhaustive reasoning
- Total functions and explicit error handling
- Pipelines, composition, and higher-order functions
- Property-based thinking and deterministic behavior
- Effect management and separation between business logic and runtime concerns
